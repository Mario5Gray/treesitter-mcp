# treesitter-mcp Deployment & Integration Notes

**Date:** 2026-02-12
**Status:** Deployed and connected to Claude Code

---

## Summary

Built and registered the `treesitter-mcp` MCP server for use within Claude Code agent sessions. This server provides tree-sitter powered code analysis tools that enable structural understanding of codebases across 10 supported languages. The goal is to use this as a shared development accelerator across all projects in `~/workspace`.

### Build & Registration

```bash
# Built the release binary
cargo build --release
# Binary: target/release/treesitter-mcp (22MB)

# Registered with Claude Code (project-local)
claude mcp add treesitter-mcp /Users/darkbit1001/workspace/treesitter-mcp/target/release/treesitter-mcp

# To register globally (all projects):
claude mcp add treesitter-mcp /Users/darkbit1001/workspace/treesitter-mcp/target/release/treesitter-mcp --scope user
```

## Available Tools

| Tool | Purpose | Token Cost |
|---|---|---|
| `view_code` | View a source file with flexible detail levels (`signatures` or `full`). Use `focus_symbol` to zoom into one symbol. | MEDIUM-HIGH |
| `code_map` | Generate hierarchical map of a directory. Start with `minimal` for large projects, then drill down. | MEDIUM |
| `find_usages` | Find ALL usages of a symbol (function, variable, class, type) across files. Semantic search, not text search. | MEDIUM-HIGH |
| `symbol_at_line` | Get symbol at a specific line with signature and scope chain. Great for stack traces. | LOW |
| `query_pattern` | Execute custom tree-sitter S-expression queries for advanced AST pattern matching. | MEDIUM |
| `parse_diff` | Analyze structural changes vs a git revision. Symbol-level diff, not line-level. | LOW-MEDIUM |
| `affected_by_diff` | Find usages affected by your changes. Shows blast radius with risk levels (HIGH/MEDIUM/LOW). | MEDIUM-HIGH |
| `type_map` | Generate usage-sorted map of all project types. Types ranked by how often they're used. | MEDIUM |
| `template_context` | Find Rust structs associated with Askama template files. | LOW-MEDIUM |

### Recommended Workflow

```
code_map (explore) --> view_code (read) --> make changes --> parse_diff (verify) --> affected_by_diff (assess risk)
```

### Supported Languages

Rust, Python, JavaScript, TypeScript, HTML, CSS, Swift, C#, Java, Go

## On-boarding Integration Guide

### How to enable treesitter-mcp for any project in `~/workspace`

There are two approaches:

#### Option A: Global registration (recommended)

Register once, available everywhere:

```bash
claude mcp add treesitter-mcp \
  /Users/darkbit1001/workspace/treesitter-mcp/target/release/treesitter-mcp \
  --scope user
```

This writes to `~/.claude.json` and makes the tools available in every Claude Code session regardless of working directory.

#### Option B: Per-project registration

Register from within the project directory:

```bash
cd ~/workspace/<project-name>
claude mcp add treesitter-mcp \
  /Users/darkbit1001/workspace/treesitter-mcp/target/release/treesitter-mcp
```

This writes to the project-local `.claude.json`.

#### Stability-Toys (`~/workspace/Stability-Toys`)

- **Language:** Python

- **treesitter-mcp coverage:** Full Python support (`*.py` files)
- **Integration steps:**
  1. Register MCP server (Option A or B above)
  2. Start a Claude Code session in the project
  3. Use `code_map` on `src/` or project root to get the lay of the land

  code_map path=~/workspace/Stability-Toys pattern="*.py" detail=minimal

### Tips for Using treesitter-mcp Across Projects

1. **Start broad, drill down.** Use `code_map` with `detail=minimal` first, then `detail=signatures`, then `view_code` on specific files.

2. **Budget your tokens.** Use `max_tokens` parameter to control output size. Start with 2000 (default), increase for large codebases.

3. **Before refactoring**, always run `find_usages` on the symbol you plan to change. This prevents breaking call sites you didn't know about.

4. **After making changes**, run `parse_diff` to verify you changed what you intended, then `affected_by_diff` to see what else might need updating.

5. **For Python projects** (Stability-Toys, tinygrad), the tools work on `*.py` files. Use the `pattern` parameter to filter: `pattern="*.py"`.

6. **For multi-language projects**, let treesitter-mcp auto-detect. It maps file extensions to parsers automatically.

7. **Prefer `find_usages` over grep** for structural queries. It understands imports, definitions, calls, and type references -- not just text matches.
